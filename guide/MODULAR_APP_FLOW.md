# Modular App Flow (New Developer Guide)

This project uses a generated app registry so mini-apps can be added/removed with minimal breakage.

## 1. Core Idea
- Mini-app folders live in `src/apps/<appName>/`.
- Only apps listed in `src/apps/registry/app-manifest.json` under `enabledApps` are included.
- Registry files are generated by `scripts/generate-app-registry.cjs`:
  - `src/apps/registry/generated/appI18nRegistry.ts` (lightweight i18n + app ids)
  - `src/apps/registry/generated/appRegistry.ts` (navigation + home widgets + types)

## 2. Daily Start
1. Run `npm run start` from project root.
2. `prestart` runs registry generation automatically.

If hot reload feels stale, run:
1. `npm run start -- --clear`

## 3. Add a New Mini-App
1. Create folder: `src/apps/<newApp>/`.
2. Add navigator:
   - `src/apps/<newApp>/navigation/<AppNavigator>.tsx`
3. Add translations (if app has UI text):
   - `src/apps/<newApp>/localization/en.ts`
   - `src/apps/<newApp>/localization/fr.ts`
4. Add app id to `enabledApps` in `src/apps/registry/app-manifest.json`.
5. If this app should be routable from shared navigation, add route metadata in `scripts/generate-app-registry.cjs` under `APP_ROUTE_META`.
6. Start app (`npm run start`) or regenerate manually (`npm run generate:app-registry`).

## 4. Remove/Disable a Mini-App
Option A: Disable without deleting code.
1. Remove app id from `enabledApps`.
2. Regenerate registry.

Option B: Remove code.
1. Delete `src/apps/<appName>/`.
2. Remove app id from `enabledApps`.
3. Regenerate registry.

## 5. Rules That Prevent Breakage
- Do not import app internals directly from shared/core files (`src/general/**`, `src/screens/**`).
- Shared/core should consume generated registries, not hardcoded app imports.
- Keep i18n imports lightweight by using `appI18nRegistry.ts` only.
- If you add/rename navigator paths, keep `APP_ROUTE_META` in sync.

## 6. Quick Troubleshooting
- Error: `Cannot read property 'en' of undefined`
  - Cause: stale/cyclic registry usage.
  - Fix: ensure i18n imports come from `appI18nRegistry.ts`, then run `npm run start -- --clear`.
- App folder exists but app not visible:
  - Check `enabledApps` in manifest.
  - Regenerate registry.
- Route missing after app add:
  - Add/update `APP_ROUTE_META` entry in generator.
  - Regenerate.
